## Q1. r90과 ρ(rho)는 무슨 의미인가?

### r90 — "에너지 90%를 설명하는 데 필요한 차원 수"

10개 태스크 벡터를 행렬(9,232,384 × 10)로 쌓고 SVD로 분해하면, 각 방향(특이벡터)이 얼마나 중요한지 특이값으로 표현된다.

```
특이값² = 그 방향이 담당하는 "에너지"
누적 에너지 합이 90%가 되는 데 필요한 방향 수 = r90
```

직관적으로:

| r90 | 의미 |
|-----|------|
| 1 | 10개 벡터가 거의 같은 방향 → 완전한 공통 subspace |
| 3 | 3개 주요 방향으로 대부분 설명 → 공통 저차원 구조 있음 |
| 9 | 9개 방향 모두 필요 → 벡터들이 각자 다른 방향 → 공통 구조 없음 |

이번 결과: r90 = **9** → 10개 벡터가 거의 독립적인 방향

### ρ(rho) — "r90을 정규화한 값"

```
ρ = r90 / min(태스크 수, √벡터 차원)
  = 9 / min(10, √9,232,384)
  = 9 / min(10, 3038)
  = 9 / 10 = 0.90
```

| ρ 값 | H1a 판정 |
|------|----------|
| < 0.30 | strong pass |
| 0.30~0.50 | marginal pass |
| 0.50~0.70 | inconclusive |
| ≥ 0.70 | fail (또는 inconclusive) |

이번 결과: ρ = **0.90** → 사실상 weak fail

## Q2. 1000번 재샘플링, CI₉₅, "10개가 너무 적어서 불안정"의 의미는?

### Bootstrap 재샘플링

"ρ=0.90이 신뢰할 수 있는 값인가?"를 검증하는 방법.  
원본 10개 태스크에서 **중복 허용**으로 10개를 뽑아 ρ를 계산하는 것을 1000번 반복한다.

```
1회차: [GSM8K, GSM8K, ARC-Easy, CommonsenseQA, ...] → ρ = 0.60
2회차: [HellaSwag, MBPP, MBPP, WinoGrande, ...]     → ρ = 0.70
3회차: [CommonsenseQA, CommonsenseQA, GSM8K, ...]    → ρ = 0.40
...
```

### CI₉₅ = [0.40, 0.70]의 의미

1000개의 ρ 값을 정렬했을 때 하위 2.5%~상위 2.5%를 제외한 범위.

```
|←2.5%→|←────────95%────────→|←2.5%→|
        0.40               0.70
```

"재샘플링을 반복하면 ρ가 0.40~0.70 사이에 95% 확률로 들어온다."  
실측 ρ=0.90은 이 범위 **밖**이다.

### "10개가 너무 적어서 불안정"의 의미

CI가 넓다(범위=0.30) = 어떤 태스크를 뽑느냐에 따라 결과가 크게 달라진다.

```
MBPP가 2번 뽑히면 → 코드 방향이 겹침 → r90 감소 → ρ 낮아짐 (0.40)
10개 모두 다르게 뽑히면 → 원본과 유사 → ρ 높음 (0.70)
```

태스크가 30~50개라면 재샘플해도 구성이 크게 안 달라지므로 CI가 좁아진다.  
10개밖에 없으니 조합에 따라 ρ가 크게 요동치는 것이다.

→ 실측값 0.90은 크지만 표본이 너무 작아 확신이 없으므로 **fail이 아닌 inconclusive**.

## Q3. r90=9여도 ρ가 낮으면 H1a를 통과할 수 있는가?

**불가능하다.** r90과 ρ는 직결돼 있다.

```
ρ = r90 / 10  (분모가 태스크 수 10으로 고정)

r90=9 이면 ρ=0.90으로 항상 고정
```

분모가 10으로 고정되는 이유:

```
SVD의 유효 특이값 최대 수 = min(행 수, 열 수) = min(9,232,384, 10) = 10
→ r90 범위: 1~10
→ 분모: min(태스크 수, √벡터 차원) = min(10, 3038) = 10
```

r90과 ρ는 같은 숫자를 스케일만 바꾼 것이다. ρ가 독립적으로 낮아질 수 없다.

### ρ가 낮아지는 두 가지 경로

| 방법 | 원리 | 진짜 의미 |
|------|------|-----------|
| r90을 낮춤 | 벡터들이 실제로 비슷한 방향 | 공통 subspace 발견 ✅ |
| 태스크 수를 30개로 늘림 | 분모가 30이 돼 ρ=9/30=0.30 | 공통 구조와 무관 ⚠️ |

**진짜 H1a 통과는 r90 자체가 낮아야 한다.**

## Q4. 후속 실험(Attention/MLP 분리, 군집 분석)의 원리는?

### 핵심 원리

> "다양성의 원천을 격리하면 숨어있던 공통 구조가 보인다."

### Plan A — Attention/MLP 분리

현재 벡터는 7개 모듈을 모두 concat한다:

```
태스크 벡터 = [Attention 파라미터 | MLP 파라미터]
              q/k/v/o_proj (23.6%) | gate/up/down_proj (76.4%)
```

**MLP가 Attention의 공통 구조를 덮는 문제**:

```
태스크 A: [Attention=방향1 | MLP=방향X]
태스크 B: [Attention=방향1 | MLP=방향Y]  ← Attention은 같지만
태스크 C: [Attention=방향1 | MLP=방향Z]    MLP가 달라서 전체 벡터는 다름

합쳐서 보면: 벡터들이 모두 다른 방향 → r90 높음 → H1a 실패
Attention만 보면: 전부 방향1 → r90=1 → H1a 강력 통과
```

**왜 Attention이 공통 구조를 가질 수 있는가**:

Attention은 "어디에 집중할지" 학습 → 포맷이 같으면 패턴이 비슷할 가능성.

```
CommonsenseQA, ARC-Challenge, ARC-Easy, OpenbookQA
→ 모두 "선택지 중 답을 찾아라" 형태
→ Attention이 학습하는 "질문-답 토큰 집중" 패턴이 유사할 수 있음
```

MLP는 "어떤 지식을 꺼낼지" 결정 → 태스크마다 완전히 다른 지식 → 독립 방향.

### Plan B — 태스크 군집 분석

10개 이질적 태스크를 혼합하면 군집 간 다양성이 공통 구조를 덮는다.

```
MCQ 4개 (비슷한 방향) ─┐
코드 2개 (비슷한 방향) ─┤ 전부 합치면 4가지 방향 → r90 높음
수학 2개 (비슷한 방향) ─┤
언어추론 2개           ─┘

MCQ 4개만 분리하면 → 비슷한 방향 → r90 = 1~2 가능 → H1a 통과
```

비유: 한국인 4명, NBA 선수 3명, 초등학생 3명의 키를 섞어서 분석하면 분산이 크지만, 한국인 4명만 보면 분산이 작다.

### 두 실험의 공통 구조

```
[현재] 전체 분석  → 다양성이 공통 구조를 덮음  → r90 높음
[Plan A] 기능 분리 → Attention 공통 구조 드러남  → r90 낮아질 수도
[Plan B] 도메인 분리 → 군집 내 공통 구조 드러남 → r90 낮아질 수도
```

### 주의: 통과해도 주장이 좁아짐

| 실험 | 통과 시 주장 |
|------|-------------|
| 전체 10개 | "모든 태스크 LoRA가 공통 subspace 공유" |
| Attention만 | "Attention 파라미터만 공통 subspace 공유" |
| MCQ 군집만 | "동일 포맷 태스크들만 공통 subspace 공유" |

더 좁은 주장이지만 **더 정밀하고 의미 있는 발견**이 될 수 있다.
